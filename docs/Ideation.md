세계 최고의 퀀트 헤지펀드 중 하나인 **Renaissance Technologies(RenTec)**의 'Real-time Trading Programmer' 포지션은 단순히 코딩을 잘하는 것을 넘어, 하드웨어와 소프트웨어의 경계에서 **극한의 성능을 뽑아낼 수 있는 엔지니어**를 찾습니다.

RenTec은 도구(Library)의 사용자보다는 **도구를 직접 만들 수 있는 사람**을 선호합니다. 지원을 위해 준비해야 할 기술 스택과 이를 증명할 수 있는 사이드 프로젝트를 정리해 드립니다.

## 1. 필수 기술 스택 (Core Tech Stack)

이 포지션의 핵심은 **저지연(Low-Latency)**과 **고신뢰성(High-Reliability)**입니다.

| 구분 | 핵심 기술 | 상세 내용 |
| --- | --- | --- |
| **언어** | **Modern C++** | C++17/20, Template Meta-programming, STL 내부 구조 이해 |
| **시스템** | **Low-level Linux** | 커널 바이패스(Kernel Bypass), 인터럽트 처리, 시스템 콜 최적화 |
| **네트워크** | **TCP/UDP/Multicast** | 소켓 프로그래밍, 이더넷 프로토콜, 고속 데이터 수신 및 파싱 |

## 2. 역량을 증명할 사이드 프로젝트 제안

단순한 웹/앱 프로젝트가 아닌, **컴퓨터 과학의 근본적인 문제**를 해결하는 프로젝트가 필요합니다.

### 프로젝트 A: 초고속 주식 호가창(Limit Order Book) 엔진 구현

금융 데이터의 핵심인 주식 호가창을 관리하는 엔진을 만듭니다. 수만 건의 매수/매도 주문을 마이크로초(μs) 단위로 처리하는 것이 목표입니다.

* **핵심 기술:** C++, Lock-free 데이터 구조, Double Buffering.
* **도전 과제:**
* `std::map`이나 `std::set` 대신 성능을 극대화할 수 있는 커스텀 데이터 구조 설계.
* 캐시 미스(Cache Miss)를 최소화하기 위한 메모리 레이아웃 최적화.
* 멀티 스레드 환경에서 데이터 경합(Race Condition) 없이 초당 수백만 건의 업데이트 처리.



### 프로젝트 B: UDP Multicast 기반 시세 수신(Market Data Feed) 핸들러

거래소에서 보내는 대량의 UDP 패킷을 유실 없이 가장 빠르게 수신하여 파싱하는 프로그램을 만듭니다.

* **핵심 기술:** 소켓 프로그래밍(Raw Sockets), 역직렬화(Deserialization).
* **도전 과제:**
* 패킷 수신 시 커널 오버헤드를 줄이기 위한 Zero-copy 기법 적용.
* SBE(Simple Binary Encoding) 또는 Flatbuffers를 활용한 초고속 파싱.
* 네트워크 지터(Jitter)를 측정하고 이를 시각화하는 도구 포함.



### 프로젝트 C: 사용자 수준의 커널 바이패스(Kernel Bypass) 실험

리눅스 커널의 네트워크 스택을 거치지 않고 사용자 공간(User Space)에서 직접 패킷을 처리하여 레이턴시를 획기적으로 줄이는 실험입니다.

* **핵심 기술:** DPDK(Data Plane Development Kit) 또는 Solarflare OpenOnload.
* **도전 과제:**
* 표준 소켓 통신과 DPDK 기반 통신의 레이턴시 차이 벤치마킹.
* CPU Pinning 및 Isolation을 통해 특정 코어가 오직 트레이딩 로직에만 전념하도록 설정.
